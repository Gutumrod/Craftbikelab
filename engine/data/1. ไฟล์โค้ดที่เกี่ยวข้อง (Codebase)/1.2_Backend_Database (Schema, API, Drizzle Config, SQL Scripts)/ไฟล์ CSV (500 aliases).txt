### 2) **Normalizer script — ตัวอย่าง (pseudo‑code + regex rules)**
**เป้าหมาย:** แปลง query ของผู้ใช้เป็น canonical tokens เพื่อแมตช์กับ alias table ได้แม่นยำขึ้น (รองรับไทย/อังกฤษ, เลขไทย, typo ชั้นต้น)

#### หลักการสั้น ๆ
1. Lowercase ทั้งหมด  
2. แปลงเลขไทย → อารบิก  
3. ลบ punctuation (.,-/\) และ extra whitespace  
4. แปลงคำย่อ/สลับสะกดเป็น canonical (mapping table)  
5. ลบ/normalize วรรณยุกต์ในบางกรณี (configurable)  
6. ทำ phonetic/fuzzy candidate generation (สำหรับ matching ชั้นถัดไป)

#### Pseudo‑code (อ่านง่ายสำหรับ dev)
```pseudo
# Normalizer main
function normalize_query(raw_query):
    q = raw_query.trim()
    q = to_lowercase(q)
    q = replace_unicode_whitespace(q, " ")
    q = convert_thai_digits_to_arabic(q)        # ๑->1, ๒->2 ...
    q = remove_punctuation(q)                   # remove . , - / ( ) etc.
    q = normalize_hyphens_and_slashes(q)
    q = collapse_multiple_spaces(q)
    q = apply_unicode_normalization(q)          # NFC/NFKC if needed
    q = map_synonyms(q)                         # e.g., "ฟอร์ซ่า" -> "forza"
    q = remove_optional_tone_marks(q)           # optional: remove diacritics for Thai
    q = expand_common_abbreviations(q)          # adv. -> adv ; nmaxx -> nmax
    q = transliterate_thai_to_roman_if_needed(q) # optional for mixed queries
    q = generate_phonetic_tokens(q)             # e.g., double metaphone / custom Thai phonetic
    return q

# Helper: convert Thai digits
function convert_thai_digits_to_arabic(s):
    mapping = {'๐':'0','๑':'1','๒':'2','๓':'3','๔':'4','๕':'5','๖':'6','๗':'7','๘':'8','๙':'9'}
    for k,v in mapping:
        s = s.replace(k, v)
    return s

# Helper: remove punctuation
function remove_punctuation(s):
    return regex_replace(s, r"[^\p{L}\p{N}\s]", " ")  # keep letters, numbers, spaces

# Helper: map synonyms (use dictionary)
SYN_MAP = {
    "ฟอร์ซ่า":"forza",
    "เอ็นแม็กซ์":"nmax",
    "เอ็นแม็ก":"nmax",
    "เวฟ":"wave",
    "เอดีวี":"adv",
    "นิมักซ์":"nmax",
    "นแม็กซ์":"nmax",
    "นแม็ก":"nmax",
    "ฟอร์ซ่า":"forza",
    "นินจา":"ninja",
    "บิ๊กไบค์":"bigbike"
    # ... load from mapping CSV
}
function map_synonyms(s):
    for key,value in SYN_MAP:
        s = regex_replace_word_boundary(s, key, value)
    return s

# Helper: generate phonetic tokens (for fuzzy)
function generate_phonetic_tokens(s):
    tokens = split_on_space(s)
    phonetic_tokens = []
    for t in tokens:
        phonetic_tokens.append(double_metaphone(t))   # for roman tokens
        phonetic_tokens.append(thai_phonetic(t))      # custom for Thai
    return join(phonetic_tokens, " ")

# Example fuzzy matching flow (matching layer)
function match_query(normalized_query):
    # 1. exact match against alias table
    if alias_table.contains(normalized_query):
        return alias_table[normalized_query]
    # 2. normalized exact (strip spaces)
    q2 = remove_spaces(normalized_query)
    if alias_table.contains(q2):
        return alias_table[q2]
    # 3. tokenized match (brand or model tokens)
    tokens = split(normalized_query)
    candidates = lookup_by_tokens(tokens)
    # 4. fuzzy score (Levenshtein / Jaro-Winkler) on candidates
    scored = []
    for c in candidates:
        score = fuzzy_score(normalized_query, c.alias)
        if score >= THRESHOLD:
            scored.append((c, score))
    sort_by_score_desc(scored)
    if top_score - second_score < AMBIGUITY_MARGIN:
        return suggest_top_k(scored, k=3)
    else:
        return scored[0].candidate
```

#### ตัวอย่าง Regex rules (ใช้ใน pipeline)
- แปลงเลขไทย:
  - `s = re.sub('[๐๑๒๓๔๕๖๗๘๙]', lambda m: thai_digit_map[m.group()], s)`
- ลบ punctuation:
  - `s = re.sub(r"[^\w\sก-๙]", " ", s, flags=re.U)`
- ลบวรรณยุกต์ (option):
  - `s = re.sub(r"[่้๊๋ะาำิีึืุู]", "", s)`  *(ปรับให้เหมาะกับการแมตช์)*
- แทนที่คำย่อ/alias:
  - `s = re.sub(r"\badv\b", "adv", s)`
  - `s = re.sub(r"\bforza\b|\bฟอร์ซ่า\b", "forza", s)`
- collapse spaces:
  - `s = re.sub(r"\s+", " ", s).strip()`

#### Fuzzy matching suggestions
- ใช้ **Levenshtein distance** หรือ **Jaro-Winkler** สำหรับคำสั้น (threshold 0.8–0.9)  
- ใช้ **phonetic matching** (Double Metaphone สำหรับโรมัน; custom Thai phonetic hashing สำหรับคำไทย) เพื่อจับเสียงใกล้เคียง  
- ให้ pipeline เป็น: **exact → normalized exact → token match → phonetic → fuzzy**

### 4) การใช้งานแนะนำ (Matching flow & thresholds)
- **Pipeline แนะนำ:**  
  1. **Normalize** (ตาม script ข้างต้น)  
  2. **Exact match** กับ alias table  
  3. **Normalized exact** (strip spaces/punctuation)  
  4. **Token match** (brand/model tokens)  
  5. **Phonetic match** (Double Metaphone + Thai phonetic)  
  6. **Fuzzy match** (Levenshtein/Jaro-Winkler) — แนะนำ threshold **0.82–0.90** สำหรับคำสั้น; ปรับตามผลจริง  
  7. **Ambiguity handling**: ถ้าคะแนน top1 ≈ top2 (difference < 0.05) ให้ **suggest top 3** แก่ผู้ใช้  
- **Fields แนะนำใน DB:** `alias,brand,model,canonical_model_key,displacement,trim,language,popularity_score`  
- **Telemetry:** เก็บ `raw_query,normalized_query,matched_alias,match_score,was_suggested,user_choice` เพื่อปรับ dataset

### 5) ข้อเสนอการทดสอบและเมตริก
- **รันชุดทดสอบ 300 queries** ตรวจดู: exact match rate, fuzzy match rate, suggestion rate, false positive rate  
- **เมตริกสำคัญ:**  
  - **Match rate** (exact + fuzzy) — เป้าหมายเริ่มต้น ≥ 90% บนชุดทดสอบ  
  - **Ambiguity rate** — % queries ที่ต้องแสดง suggestion  
  - **Suggestion CTR** — % ที่ผู้ใช้เลือก suggestion  
  - **False positive rate** — เป้าหมาย ≤ 2–3% (ปรับ threshold)
